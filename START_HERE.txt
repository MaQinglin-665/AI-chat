Xinyu Desktop Pet v1.4 Preview - Start Here
===========================================

This is a source-based preview for early testers and contributors. It is not a
finished Windows installer.

Fastest guided path
-------------------

Double-click:

    install_and_start.bat

This guided first-run entry prepares dependencies, applies the preview profile,
helps you configure your own LLM, runs a first-chat smoke check, then starts the
Electron desktop pet.

Important notes:

- You still need your own model/provider/API key, or a local Ollama model.
- The smoke check sends one small request to your configured model.
- Desktop observation, screenshots, user file reading, tool calling, and shell
  execution stay disabled by default.

Step-by-step path
-----------------

If you want more control or easier troubleshooting, run:

    prepare_preview_environment.bat
    powershell -NoProfile -ExecutionPolicy Bypass -File scripts\configure-llm.ps1
    powershell -NoProfile -ExecutionPolicy Bypass -File scripts\diagnose-llm-link.ps1 -SoftFail
    powershell -NoProfile -ExecutionPolicy Bypass -File scripts\first_chat_smoke.ps1
    start_electron.bat

Model choice
------------

Xinyu does not include a cloud model, hosted endpoint, or API key. You choose
the provider/model. See:

    docs\model-selection.md

If model probe success is below 80%, or normal turns often take more than 15
seconds, switch to a faster stable model before judging the character
experience.

First things to try
-------------------

- Send a short English chat message.
- Check that the Live2D model is visible.
- Check that browser TTS speaks the full reply.
- Try /doctor for local diagnostics.
- Try /braindebug after a normal chat turn.
- Watch whether spontaneous replies and small stage reactions appear without
  becoming noisy.
