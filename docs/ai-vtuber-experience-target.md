# AI VTuber Experience Target

本文定义项目长期想接近的体验目标。它可以受 AI VTuber / Neuro-sama-like 互动方向启发，但不是复制、克隆或承诺当前阶段具备完整自动直播能力。

项目当前仍是 MVP / 开源孵化阶段。我们的方向是：先做一个在 Windows 桌面上低打扰、可陪伴、会表达的 AI 角色，再逐步增强可观察性、语音互动和安全的上下文能力。

## One-Sentence Target

做一个会说话、会表达、能长期陪伴、但不会越界打扰用户的桌面 AI VTuber / AI pet。

## What This Is Not

- 不是普通聊天窗口换一层 Live2D 皮肤。
- 不是成熟商业 AI 助理。
- 不是默认自动直播机器人。
- 不是默认会看屏幕、读文件、执行命令的桌面代理。
- 不是 Neuro-sama 的直接复制。

## Experience Pillars

### 1. Character Continuity

用户应该感觉自己在和同一个角色互动，而不是每一轮都在调用一个新的 chatbot。

目标表现：

- 名字、人设、称呼、语气稳定。
- 多轮对话中的情绪和关系变化有连续性。
- 记忆使用克制、可解释，不把无关旧信息强塞进回复。
- 角色有轻微个性和偏好，但不压过用户需求。

验收信号：

- 同一场会话中，欢迎、闲聊、安慰、鼓励、结束语的风格属于同一个角色。
- 用户问“你是谁”时，角色介绍一致，不频繁自称 ChatGPT 或泛用 AI。
- 角色不会因为一次失败就变成系统错误播报器。

### 2. Unified Multimodal Expression

文字、TTS、字幕、表情和动作应该表达同一个状态。

目标表现：

- 开心时语言轻快，TTS 稍有活力，Live2D 有轻微开心反馈。
- 安慰时语言收敛，TTS 放缓，动作温和。
- 思考时可以有停顿、短句和思考动作，但不长时间卡死。
- 提醒时短而明确，动作像确认而不是兴奋表演。

验收信号：

- 不出现“文字在安慰、动作却过度兴奋”的割裂感。
- 不把 `emotion`、`action`、`voice_style`、JSON 等内部字段显示或读出。
- TTS 结束后口型和动作能回到 idle。

### 3. Real-Time Interaction Feel

用户应该觉得角色“正在回应我”，而不是把表单提交给后端。

目标表现：

- 文本输入后尽快出现可见反馈。
- TTS 首音延迟可接受。
- 长句 TTS 不丢尾，不突然静音。
- 语音输入能完成一轮短对话，不重复发送同一段 ASR。
- 前端或后端失败时，有可读的恢复路径。

验收信号：

- 一条短输入能完成：用户输入 -> 回复文本 -> TTS -> 字幕 / Live2D 反馈。
- 语音输入失败时，用户还能用键盘继续聊天。
- GPT-SoVITS 不可用时，browser TTS 或纯文本体验仍可继续。

### 4. Watchable Demo Quality

项目应该能录出短 demo，让旁观者看出“这是一个角色”，而不是接口测试。

目标表现：

- 有固定 demo 场景：欢迎、闲聊、安慰、提醒、鼓励、语音对话、动作反馈、结束语。
- 每个场景有稳定输入、期望语言风格、TTS 表现、Live2D 表现和失败回退。
- Demo 不依赖私人桌面、真实 API Key 或不可复现的本地状态。

验收信号：

- 30-60 秒内可以展示角色说话、动作、字幕和文本互动。
- 2-3 分钟内可以展示至少三个不同情绪场景。
- 失败场景也能被解释成 MVP 可恢复限制，而不是应用失控。

### 5. Low-Interruption Companionship

角色可以有陪伴感，但不能变成频繁打断用户的桌面弹窗。

目标表现：

- 主动行为有冷却时间、场景门槛和关闭路径。
- 轻量提醒和关心优先于长篇打扰。
- 用户沉默时，不默认假设需要介入。
- 主动行为必须能解释“为什么现在出现”。

验收信号：

- 默认配置不自动开启主动观察或自动聊天。
- 用户可以关闭主动功能。
- 主动消息不会连续刷屏。

### 6. Safety Trust

越像“活着”的角色，越要让用户确信它不会乱碰电脑。

目标表现：

- 默认不观察桌面。
- 默认不读取用户文件。
- 默认不执行 shell。
- 默认不启用工具调用。
- 高风险能力必须显式 opt-in，并且有确认、日志、退出和回滚路径。

验收信号：

- README、setup、QA、release 文档都不鼓励用户放宽安全默认值。
- Demo 不展示或暗示默认自动桌面代理能力。
- `/api/health`、`/doctor` 等诊断不泄露 token、key 或私人内容。

## Product Ladder

### v1.2: First Run & Stability

先让新用户能跑起来，并能定位失败原因。

核心体验：

- 文本聊天可用。
- Live2D 能显示。
- Browser TTS 可作为基础语音路径。
- 常见配置错误可诊断。

### v1.3: Character Runtime

让角色状态、情绪、动作和记忆有更清晰的运行时契约。

核心体验：

- 角色状态可追踪。
- 情绪 / 动作 metadata 不泄露给用户。
- 主动陪伴能力保持 opt-in 和 fail-closed。

### v1.4: AI VTuber Feeling

把语言、声音、动作和 demo 场景揉成一个可感知角色体验。

核心体验：

- 角色风格稳定。
- TTS 和动作反馈一致。
- 有可复现 demo 场景。
- 失败时主流程不中断。

### v1.5: Desktop Awareness

在用户授权下引入安全的桌面上下文辅助。

核心体验：

- 手动触发优先。
- 最小权限和可关闭。
- 不默认截图，不默认读取文件。

### v2.0: Productized Release

把发布、质量门槛、配置迁移和扩展接口稳定下来。

核心体验：

- 可重复发布。
- 核心链路稳定。
- 外部贡献者能按文档扩展。

## Demo North Star

一个理想短 demo 应该让观众看到：

1. 桌面上有一个 Live2D 角色。
2. 用户发起一句自然输入。
3. 角色以稳定人设回应。
4. TTS、字幕、表情和动作同步表达同一情绪。
5. 出现一个轻量主动或提醒场景，但不打扰。
6. 诊断或失败回退可解释、可恢复。
7. 全程不展示私人密钥、桌面隐私或高风险默认权限。

## Design Guardrails

- 角色感优先于功能堆叠。
- 稳定体验优先于炫技。
- 低打扰优先于主动刷存在感。
- 可解释和可关闭优先于自动化。
- 安全默认值优先于 demo 便利。
