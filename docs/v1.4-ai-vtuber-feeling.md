# v1.4 AI VTuber Feeling

本页是 v1.4 阶段的体验规格草案，用来准备角色体验、demo 场景和人工验收标准。它不要求现在改动前端运行时，也不改变安全默认值。

v1.4 的目标不是把项目包装成成熟商业产品，而是在现有 MVP 基础上，让用户更明显地感到“桌面上有一个角色在陪伴和回应我”。

更长期的 AI VTuber / 桌宠体验目标见 `docs/ai-vtuber-experience-target.md`。

## Experience Goal

v1.4 关注四件事：

- 语言：回复更像稳定角色，而不是泛用聊天机器人。
- 声音：TTS 节奏、情绪和长句表现更可控。
- 动作：Live2D 表情 / motion 与文字和声音表达一致。
- 回退：LLM、TTS、ASR 或动作失败时，文本聊天主流程仍可继续。

## Non-Goals

- 不默认启用桌面自动观察。
- 不默认启用工具调用、文件读取或 shell 执行。
- 不把 GPT-SoVITS 作为首次运行必需服务。
- 不要求用户为了 demo 暴露 API Key、私人桌面或敏感日志。
- 不承诺 Neuro-sama 级别的完整 AI VTuber 自动直播能力。

## Acceptance Standards

### Language Style

- [ ] 同一会话中，角色称呼、语气、句长和互动方式保持一致。
- [ ] 角色可以表达轻微情绪，但不应夸张、刷屏或频繁自称 AI。
- [ ] 安慰、鼓励、提醒等场景有不同语气，不只是同一套模板换词。
- [ ] 回复默认短而自然；长句只在用户请求或 demo 场景需要时出现。
- [ ] 不把内部字段、JSON、emotion/action metadata 当作普通文本说出或显示。

### TTS And Voice

- [ ] Browser TTS 作为基础回退路径可用。
- [ ] GPT-SoVITS 可作为高级本地体验，但服务不可用时文本聊天仍可用。
- [ ] 短句首音不应出现明显延迟。
- [ ] 长句应能播放完整，不丢最后一句。
- [ ] 情绪化回复的语速、停顿或音量变化应温和，不刺耳、不忽大忽小。
- [ ] TTS 失败时有可读提示或 debug 信息，不让用户以为整个应用坏了。

### Live2D Motion And Expression

- [ ] 欢迎、开心、安慰、思考、提醒、结束语等场景有可见差异。
- [ ] 动作触发频率不过高，不应每个字都触发大动作。
- [ ] 表情 / motion 与回复情绪一致，例如安慰时不使用过度兴奋动作。
- [ ] TTS 播放时口型 / 身体反馈和声音大致同步，结束后能回到 idle。
- [ ] 动作失败不影响文本聊天和 TTS 主流程。

### Failure Fallback

- [ ] LLM 失败时给出可理解的错误摘要和下一步建议。
- [ ] TTS 失败时文本仍显示，用户可以继续聊天。
- [ ] ASR 失败时键盘输入仍可用。
- [ ] Live2D motion 失败时角色显示和聊天不应崩溃。
- [ ] `/doctor`、`/ttsdebug`、`/translatedebug`、`/memorydebug`、`/micdebug` 能辅助定位。

### Safety And Privacy

- [ ] 默认不观察桌面。
- [ ] 默认不自动读取文件。
- [ ] 默认不执行 shell。
- [ ] 默认不启用工具调用。
- [ ] Demo 素材不展示真实 API Key、Token、私人桌面内容或敏感日志。

## Reproducible Demo Scenes

每个场景都应记录：配置、输入、期望文字风格、TTS 期望、Live2D 期望、失败回退。

### 1. Welcome

输入：

```text
你好，今天第一次启动，跟我打个招呼。
```

期望：

- 文字：简短欢迎，说明自己是桌面伙伴，不夸大能力。
- TTS：语气轻快，首音延迟低。
- Live2D：轻微开心 / 招呼动作。
- 回退：TTS 不可用时仍显示文本欢迎。

### 2. Casual Chat

输入：

```text
我今天有点不知道先做什么。
```

期望：

- 文字：先共情，再给一个小建议。
- TTS：稳定自然，不像客服话术。
- Live2D：思考或温和回应动作。
- 回退：LLM 慢时 UI 不应冻结。

### 3. Comfort

输入：

```text
我刚刚被否定了，有点难受。
```

期望：

- 文字：先接住情绪，不急着讲大道理。
- TTS：语速略慢，音量稳定。
- Live2D：安慰 / calm 表情，不使用兴奋动作。
- 回退：禁止输出夸张心理诊断或冒充专业治疗。

### 4. Reminder

输入：

```text
提醒我十分钟后休息一下眼睛。
```

期望：

- 文字：确认提醒内容和时间。
- TTS：简洁确认。
- Live2D：轻微点头或确认动作。
- 回退：提醒能力不可用时说明原因和替代方案。

### 5. Encouragement

输入：

```text
我写代码卡住了，给我一点鼓励。
```

期望：

- 文字：短句鼓励，不灌鸡汤。
- TTS：温暖、有一点能量。
- Live2D：轻微开心或加油动作。
- 回退：TTS 失败时文字仍保持角色感。

### 6. Long Sentence Voice

输入：

```text
请用稍微完整一点的话，告诉我为什么今天先完成一个小任务也算是进展。
```

期望：

- 文字：允许比平时长，但不要大段说教。
- TTS：长句完整播放，不丢尾句。
- Live2D：口型持续到语音结束附近，结束后回 idle。
- 回退：GPT-SoVITS 不可用时可走 Browser TTS 或只显示文本。

### 7. Failure Diagnostics

输入：

```text
/doctor
```

期望：

- 文字：自检结果面向普通用户，能说明下一步。
- TTS：不强制朗读长诊断。
- Live2D：不需要大动作，保持稳定。
- 回退：LLM/TTS 某项失败时不要让整个自检卡死。

### 8. Closing

输入：

```text
我准备下线了，跟我说句收尾的话。
```

期望：

- 文字：简短收尾，保持陪伴感。
- TTS：平稳、轻柔。
- Live2D：挥手或轻微告别动作。
- 回退：语音或动作失败时文本仍可用。

## Manual QA Matrix

| Area | Must Check |
| --- | --- |
| Text | 风格一致、无内部 metadata 泄漏、错误提示可读 |
| TTS | Browser fallback、GPT-SoVITS 长句、失败释放状态 |
| Live2D | idle、tap、talk gesture、情绪动作、拖拽时暂停 |
| ASR | 麦克风开关、低音量提示、失败后键盘可用 |
| Safety | 默认不观察桌面、不工具调用、不 shell、不读文件 |
| Demo | 不暴露密钥、私人桌面或敏感日志 |

## Recording Notes

- 使用干净配置和非敏感桌面录制。
- 录制前跑 `/doctor` 和一次语音测试。
- 每个 demo 场景保留输入文本、配置摘要和失败回退说明。
- 如果使用 GPT-SoVITS，说明它是高级本地体验，不是首次运行必需项。
- 如果 Live2D 模型或动作资源来自第三方，确认许可和署名要求。
