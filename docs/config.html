    发布指引

    1. 先在本页执行一次“配置连通性自检”，确保关键接口状态正常。
    2. 将首页下载链接与仓库地址指向当前 GitHub 仓库与 Releases。
    3. 发布前在本地运行桌宠并验证“一键应用配置”流程可走通。
    4. 如验证中配置异常，可用“回滚到最近备份”快速恢复后再继续。
    5. 在 GitHub 创建新版本 Release，并上传安装包与更新说明。
