# 项目开发路线图（ROADMAP）

说明：
- 本路线图用于规划后续迭代，不代表当前已全部实现。
- 每个条目包含用户价值、可能涉及文件和验收标准，便于直接排期与验收。

## 第一阶段：稳定可用

| 功能 | 用户价值 | 可能涉及文件 | 验收标准 |
| --- | --- | --- | --- |
| 首次启动向导 | 新用户第一次打开就知道“先配什么、怎么测通”，降低放弃率。 | `web/index.html`、`web/app.js`、`docs/config.html`、`docs/js/config-page.js`、`config.example.json` | 首次运行自动弹出向导；完成后生成可用配置；向导内能完成一次 LLM 连通性检测；可手动再次打开向导。 |
| 配置检测 | 在聊天前提前发现配置错误，避免“发消息后才报错”。 | `config.py`、`app.py`、`docs/js/config-page.js`、`web/app.js` | 支持一键检测 LLM/TTS/模型路径/端口；检测结果分“通过/警告/失败”；失败项给出可执行修复建议。 |
| 错误提示优化 | 用户能看懂错误并知道下一步，减少“卡住”。 | `app.py`、`llm_client.py`、`tts.py`、`web/app.js`、`web/index.html` | 常见错误（鉴权、超时、服务未启动、模型不存在）有明确中文提示；前端不展示堆栈原文；提示中包含“重试/去配置页”入口。 |
| 日志导出 | 出问题时用户可一键打包日志，便于定位和反馈。 | `electron/main.js`、`app.py`、`web/app.js`、`docs/config.html` | 支持导出最近 N 天日志为单个压缩包；默认脱敏（API Key、Token）；导出后提示保存位置。 |
| 一键重置配置 | 配错后能快速回到可启动状态，不需要手动删文件。 | `config.py`、`docs/js/config-page.js`、`docs/config.html`、`config.example.json` | 提供“重置为默认（保留备份）”；执行前二次确认；重置后可重新加载并正常启动。 |

## 第二阶段：有陪伴感

| 功能 | 用户价值 | 可能涉及文件 | 验收标准 |
| --- | --- | --- | --- |
| 人设卡管理 | 用户可切换角色风格，提升长期新鲜感。 | `memory.py`、`app.py`、`web/app.js`、`web/index.html`、`docs/config.html` | 支持创建/编辑/切换人设卡；切换后新对话即时生效；旧会话不被破坏。 |
| 主动问候 | 桌宠可在合适时机打招呼，增强存在感。 | `web/app.js`、`memory.py`、`app.py`、`config.py` | 支持按时间段与冷却时间触发；尊重“低打扰”开关；连续触发有去重机制。 |
| 情绪状态 | 回复语气和动画状态更连贯，不再每句都同一种情绪。 | `memory.py`、`web/app.js`、`app.py`、`web/index.html` | 情绪状态可读可写；情绪变化有边界（不突变）；对话与 Live2D 表现存在对应关系。 |
| 亲密度 | 长期互动有积累反馈，提高陪伴驱动力。 | `memory.py`、`app.py`、`web/app.js`、`config.py` | 亲密度有明确增减规则；可在界面查看当前等级；不因重启丢失。 |
| 记忆可视化 | 用户知道“它记住了什么”，可建立信任并可控。 | `memory.py`、`web/index.html`、`web/app.js`、`app.py` | 能查看近期记忆摘要与来源；支持删除单条记忆；删除后后续提示词不再引用该条。 |

## 第三阶段：有生产力

| 功能 | 用户价值 | 可能涉及文件 | 验收标准 |
| --- | --- | --- | --- |
| 截图分析 | 用户可直接让桌宠解释屏幕内容，减少切换工具。 | `electron/main.js`、`electron/preload.js`、`app.py`、`llm_client.py`、`web/app.js` | 可从界面触发截图并发送分析请求；用户可预览后再发送；失败时给出权限与重试提示。 |
| 番茄钟 | 提升专注并形成节奏提醒。 | `web/app.js`、`web/index.html`、`memory.py`、`config.py` | 支持开始/暂停/结束；默认 25/5 可配置；阶段切换有文本提醒和可选语音提醒。 |
| 任务提醒 | 将临时待办转为可执行提醒，减少遗忘。 | `web/app.js`、`app.py`、`memory.py`、`config.py` | 支持添加/编辑/完成任务；到时触发提醒；重启后提醒数据仍在。 |
| 学习助手 | 对学习场景给出结构化帮助（总结、提问、复习）。 | `app.py`、`memory.py`、`tools.py`、`web/app.js` | 提供固定学习模式入口；可生成“今日学习回顾”；用户可关闭该模式避免打扰。 |
| 代码助手 | 在开发场景下提供轻量解释与排错建议。 | `app.py`、`tools.py`、`web/app.js`、`config.py` | 支持粘贴代码并获取解释/修改建议；默认不开启危险 shell 执行；工具调用失败可回退纯文本回答。 |

## 第四阶段：社区化

| 功能 | 用户价值 | 可能涉及文件 | 验收标准 |
| --- | --- | --- | --- |
| 人设卡导入导出 | 用户可分享角色设定，快速复用。 | `memory.py`、`app.py`、`web/app.js`、`web/index.html` | 支持本地文件导入/导出；导入前做字段校验；失败项有明确提示。 |
| 语音包 | 用户可切换声音风格，提高个性化。 | `tts.py`、`config.py`、`web/app.js`、`docs/config.html` | 可在配置中切换语音包；切换后下一句生效；不影响已有 TTS provider 兼容。 |
| Live2D 皮肤 | 提供外观扩展能力，增强可玩性。 | `web/`、`docs/live2d/`、`config.py`、`web/app.js` | 支持添加并切换皮肤；资源缺失时不崩溃并回退默认皮肤；移动端与桌面端均可正常显示。 |
| 插件系统 | 允许社区扩展能力，减少核心仓库耦合。 | `tools.py`、`app.py`、`config.py`、`web/app.js`、`electron/preload.js` | 有明确插件清单与启停机制；插件权限可控；单个插件异常不影响主流程。 |
| 分享配置 | 用户可分享可复用配置模板，降低新手门槛。 | `docs/config.html`、`docs/js/config-page.js`、`config.py`、`config.example.json` | 支持导出“脱敏配置”；导入时有兼容检查；导入后可一键验证并回滚。 |

## 最推荐马上做的 5 个任务

1. 首次启动向导（直接降低新用户流失，投入小、收益高）。
2. 配置检测（把失败前置到“可修复”阶段）。
3. 错误提示优化（把技术错误转成用户可执行步骤）。
4. 一键重置配置（救场能力强，减少群里重复答疑）。
5. 主动问候（在“低打扰”前提下最快提升陪伴感）。
