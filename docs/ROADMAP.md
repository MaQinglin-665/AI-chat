# Roadmap (v1.2 -> v2.0)

> 说明：这是开源可体验版的产品路线，用于对齐阶段目标与验收标准，不代表所有项已完成。

## Product Direction

项目主线是：先把“稳定可陪伴的桌面角色”做好，再逐步扩展桌面感知与工具能力。

当前阶段不追求全能助手，也不默认开启高风险能力。优先级是：

1. 稳定首次运行、文本聊天、TTS、ASR。
2. 让 Live2D、语气、情绪、动作形成可感知的角色反馈。
3. 在用户明确授权后，再提供桌面上下文辅助。
4. 工具调用、shell、文件访问保持 opt-in，并保留确认和退出路径。

## Immediate Focus (2026-05)

- 收口语音输入：麦克风权限、音量阈值、ASR 请求、低音量提示、`/micdebug` 排查。
- 补齐语音排障文档：Vosk 模型、浏览器权限、TTS 回退、常见错误码。
- 修整公开仓库第一印象：README、快速启动、截图/GIF、当前限制说明。
- 评估并修复用户可见中文乱码，优先处理首页、配置、启动帮助、发布清单。
- 继续保持安全默认值：不自动观察桌面，不默认启用工具调用，不默认执行 shell。

## v1.2 First Run & Stability

### Goals

- 新用户能在较短时间内完成首次可用体验。
- 常见失败场景可诊断、可恢复，不轻易“卡死”。
- 公开仓库门面和基础流程可被外部开发者快速理解。
- 文本聊天、TTS、ASR 三条核心交互链路都有清晰排障路径。

### Major Tasks

- 首次启动引导与默认配置链路优化（LLM/TTS/模型路径检查）。
- 配置检测与错误提示统一（失败原因 + 修复建议）。
- 启动与健康检查流程增强（可观测、可快速判断状态）。
- 文档入口统一（README / CONTRIBUTING / CHANGELOG / SECURITY）。
- 语音输入稳定化（ASR 阈值、低音量检测、API token、Vosk 模型提示）。
- TTS 稳定化（超时、重试、浏览器 TTS 回退、字幕/状态提示）。
- 用户可见乱码清理（README、docs、前端首屏和配置页优先）。

### Acceptance Criteria

- 首次启动引导完成后可稳定进入对话。
- 常见配置错误可在 UI 或日志中看到明确修复建议。
- 关键健康检查接口与本地调试流程可复现。
- 社区贡献者可根据文档完成本地运行和基础验证。
- 连续运行 30 分钟时，文本聊天、Live2D 显示、默认 TTS 不应阻塞主流程。
- ASR 或 TTS 不可用时，用户能从文档和 UI 提示定位下一步。
- 发布前检查项见 `docs/release-v1.2-checklist.md`。

## v1.3 Character Runtime

### Goals

- 角色运行时能力更完整，角色行为不再“只会回答问题”。
- 角色状态（情绪、记忆、陪伴策略）在多轮对话中更连贯。
- 项目命名、角色名、人设配置在 README、UI、配置文件中保持一致。

### Major Tasks

- 角色状态机与运行时上下文管理。
- 情绪状态与动作映射增强（语音、文本、动作联动）。
- 低打扰主动交互策略（冷却时间、场景门限、可关闭）。
- 角色记忆读写策略优化（可控、可清理、可解释）。
- 人设卡配置简化（普通用户可理解，可恢复默认值）。
- `emotion` / `action` / `voice_style` 的前后端契约固化。

### Acceptance Criteria

- 同一会话中角色状态变化自然且可追踪。
- 主动交互不会高频打断用户，且可一键关闭。
- 记忆变更有清晰边界，不出现明显错误注入。
- 一轮对话后，文字、声音、动作能表达一致的角色状态。
- v1.3 开工计划见 `docs/v1.3-character-runtime-plan.md`。

## v1.4 AI VTuber Feeling

### Goals

- 明显提升“像在和一个角色互动”的主观感受。
- 声音、语言风格、动作节奏之间的一致性增强。
- 形成可录制、可演示、可复现的核心体验场景。

### Major Tasks

- 对话输出风格控制（角色语气、节奏、口癖约束）。
- TTS 表现力增强与失败回退优化。
- Live2D 表情/动作触发策略精细化。
- 关键互动场景脚本化（欢迎、提醒、鼓励、收尾）。
- Demo 场景整理（闲聊、安慰、提醒、语音对话、动作反馈）。

### Acceptance Criteria

- 连续对话中角色风格一致性明显提升。
- TTS 失败时可平滑降级，不中断主流程。
- 关键场景的动作与语音反馈有可见一致性。
- Release 页面能用截图/GIF/短视频说明项目卖点。

## v1.5 Desktop Awareness

### Goals

- 在用户授权前提下，具备基础桌面上下文感知与辅助能力。
- 保持隐私最小化和低打扰原则，不扩张默认权限。

### Major Tasks

- 桌面上下文采集入口治理（手动触发优先，显式开关）。
- 截图/内容理解链路稳定化与脱敏策略。
- 与提醒、任务、陪伴策略联动（基于上下文但不过度打断）。
- 数据生命周期管理（缓存、清理、可见性）。
- 权限中心统一管理截图、文件、shell、工具调用等高风险能力。

### Acceptance Criteria

- 桌面感知能力默认安全、默认最小权限。
- 上下文分析链路可稳定运行并可快速关闭。
- 隐私相关提示、开关和数据清理路径可达可用。
- 不默认自动观察桌面，不默认读取用户文件，不默认执行 shell。

## v2.0 Productized Release

### Goals

- 从“可体验项目”升级为“可持续维护的产品化版本”。
- 形成稳定发布节奏、质量基线与可扩展生态接口。

### Major Tasks

- 发布工程化（版本规范、打包流程、发布清单、回滚策略）。
- 质量门禁（测试覆盖、回归清单、稳定性基线指标）。
- 配置与升级策略（兼容迁移、默认值治理、变更提示）。
- 生态准备（插件接口、文档体系、贡献流程成熟化）。
- 配置中心替代大部分手改 JSON 的路径。

### Acceptance Criteria

- 可重复执行的发布流程与版本节奏。
- 核心链路在目标环境下达到稳定运行标准。
- 升级时配置兼容与变更提示可控。
- 外部贡献者能在文档指导下完成扩展与提交流程。

## Out of Scope (Current Cycle)

- 不在当前周期内引入大规模业务重构。
- 不在当前周期内默认放宽安全能力。
- 不在当前周期内承诺商业化 SLA。
- 不在语音和角色体验稳定前，把工具调用作为主卖点。
